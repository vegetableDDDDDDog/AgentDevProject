### 1. Task 7: 登录页提示增强

#### 问题描述
用户会话过期后，需要重新登录，但登录页没有友好的提示信息，用户体验不佳。

#### 解决方案
在登录页添加 URL 参数检查，根据不同的登出原因显示相应的提示信息：
- `reason=session_expired`: "您的会话已过期，请重新登录"
- `reason=token_invalid`: "登录状态无效，请重新登录"
- `reason=logout`: "您已成功退出登录"

#### 相关代码
**frontend/src/pages/LoginPage.tsx**:
```typescript
import { useSearchParams } from 'react-router-dom';

// 检查 URL 参数中的提示信息
useEffect(() => {
  const reason = searchParams.get('reason');
  if (reason === 'session_expired') {
    setInfoMessage('您的会话已过期，请重新登录');
  } else if (reason === 'token_invalid') {
    setInfoMessage('登录状态无效，请重新登录');
  } else if (reason === 'logout') {
    setInfoMessage('您已成功退出登录');
  }
}, [searchParams]);
```

**frontend/src/services/auth.ts**:
```typescript
/**
 * 强制登出（清除所有认证信息）
 *
 * @param reason - 登出原因，用于在登录页显示提示
 */
export const forceLogout = (reason: 'session_expired' | 'token_invalid' = 'session_expired'): void => {
  removeToken();
  window.location.href = `/login?reason=${reason}`;
};
```

#### 测试结果
- ✅ 代码实现完成

- ✅ forceLogout() 正确添加 reason 参数

## 🎓 关键知识点

### React Router URL 参数处理

**知识点**: 使用 `useSearchParams` 读取 URL 查询参数

**代码示例**:
```typescript
import { useSearchParams } from 'react-router-dom';

const [searchParams] = useSearchParams();
const reason = searchParams.get('reason');
```

**注意事项**:
- `useSearchParams` 类似于 `useState`，返回数组的第二个元素是更新函数
- `get()` 方法返回 `string | null`
- 需要检查 `null` 值

---

### TypeScript 函数参数默认值

**知识点**: 为函数参数提供类型和默认值

**代码示例**:
```typescript
export const forceLogout = (
  reason: 'session_expired' | 'token_invalid' = 'session_expired'
): void => {
  // ...
};
```

**注意事项**:
- 使用联合类型限制参数范围
- 默认值必须是类型之一
- 有助于防止拼写错误

### 关键流程

**主动刷新流程**:
```
1. 请求拦截器检查: isTokenExpiring() → true
2. 加刷新锁: isRefreshing = true
3. 调用 refreshAccessToken()
4. 更新 token 和 expires_at
5. 释放刷新锁: isRefreshing = false
6. 处理队列: processQueue()
```

**被动刷新流程**:
```
1. 收到 401 响应
2. 检查 _retry 标记（避免无限循环）
3. 加刷新锁: isRefreshing = true
4. 调用 refreshAccessToken()
5. 更新 token 和 expires_at
6. 重试原请求
7. 失败则 forceLogout()
```
