### 4. 租户 LLM 配置自动同步

#### 问题描述
每次重建数据库后，需要手动配置租户的 LLM API Key 和 Base URL，导致聊天功能报错。

#### 解决方案
在 `api/main.py` 的 `lifespan` 启动函数中添加自动配置同步：
```python
# 自动同步全局配置到所有租户
if global_api_key and global_base_url:
    for tenant in tenants:
        if not tenant.settings:
            tenant.settings = {}
        if not tenant.settings.get('llm_api_key') or not tenant.settings.get('llm_base_url'):
            tenant.settings['llm_api_key'] = global_api_key
            tenant.settings['llm_base_url'] = global_base_url
            updated_count += 1
```

#### 测试结果
✅ 服务重启后自动为租户配置 LLM，聊天功能正常

### 6. 删除文档功能修复

#### 问题描述
删除文档时报错 `NOT NULL constraint failed: document_processing_tasks.document_id`

#### 根本原因
删除文档时，关联的 `DocumentProcessingTask` 记录没有先删除，导致外键约束冲突

#### 解决方案
在 `knowledge.py:243-282` 的 `delete_document` 函数中，先删除关联任务：
```python
# 先删除关联的任务记录
from services.database import DocumentProcessingTask
tasks = db.query(DocumentProcessingTask).filter(
    DocumentProcessingTask.document_id == document_id
).all()
for task in tasks:
    db.delete(task)
```

#### 相关文件
- `api/routers/knowledge.py:243-282`

#### 测试结果
✅ 文档删除成功，统计正确更新

## 🎓 关键知识点

### 智谱 AI 模型配置

**Embedding 模型**: 使用 `embedding-2`，不是 `embedding-3`
**LLM 模型**: 使用 `glm-4.7` 或其他支持的模型

**环境变量配置**:
```bash
# 正确的配置
OPENAI_API_KEY=xxx
OPENAI_API_BASE=https://open.bigmodel.cn/api/paas/v4/
MODEL_NAME=glm-4.7           # 注意：不是 OPENAI_MODEL
EMBEDDING_MODEL=embedding-2  # 注意：不是 OPENAI_EMBEDDING_MODEL
```

### SQLAlchemy 外键约束处理

**问题**: 删除有外键关联的记录时会报错
**解决方案**: 先删除关联的子记录，再删除父记录

**代码示例**:
```python
# 错误方式
db.delete(parent)  # 如果有子记录的外键约束为 NOT NULL，会报错

# 正确方式
for child in parent.children:
    db.delete(child)
db.delete(parent)
db.commit()
```

### 前端状态同步

**问题**: 删除/更新操作后，相关组件的状态未更新
**解决方案**: 在回调函数中触发列表刷新

**代码示例**:
```typescript
const handleDelete = async (id: string) => {
  await deleteItem(id);
  message.success('删除成功');

  // 刷新列表以更新 UI
  await loadList();  // ← 关键
};
```
