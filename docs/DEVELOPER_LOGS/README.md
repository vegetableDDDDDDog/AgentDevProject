# 开发者日志索引

本目录记录项目开发过程中的日常工作日志。

## 📁 目录结构

```
DEVELOPER_LOGS/
├── 2026/
│   ├── 2026-02-28.md
│   └── ...
├── TEMPLATE.md
└── README.md (本文件)
```

## 📝 如何使用

### 1. 创建新日志

复制 `TEMPLATE.md`，重命名为 `YYYY-MM-DD.md`，放在对应年份目录下。

### 2. 日志命名规范

- 文件名：`YYYY-MM-DD.md` (例如: 2026-02-28.md)
- 目录：按年份组织 (例如: 2026/)

### 3. 日志内容

每篇日志应包含：
- 📋 基本信息（日期、开发者、任务）
- 🎯 完成的工作
- 🐛 修复的 Bug
- 📝 修改的文件
- 💡 经验总结
- 🚀 下一步计划
